How does Blockchain work in cryptocurrencies?

By Akhil Karra, November 27th 2018

A blockchain of a cryptocurrency is simply a ledger that tracks who has how much of cryptocurrency. But how do they really work?

The transaction starts with a wallet with a "private key." This transaction is then grouped with other transactions securly into a "block." The private key serves as proof to show that the transaction came from the owner of the wallet, not from anyone else (Kharpal). When the transactions are grouped, a key is made from a hash function (a binary mathematical function that makes a pseudo-random output) for the entire block, which serves as the identifier for that block (Karra). 

Then, the specialized computers of the various miners compete complex mathematical problems for the transaction to go through, which get added to the chain of transactions.  These computers compete to solve complex mathematical problems for the transaction to continue. The first one to solve the problems is awarded cryptocurrency. The block then joins the other blocks in the chain, hence the name "blockchain" (Kharpal). Although the mathematical problems themselves do not contribute to the security of the transaction, the cost of electricity and cooling of the computers that execute these problems contributes to the overall value of the cryptocurrency itself.  

The part of the blockchain that makes it impenitratible is the key for each block. Firstly, a particular block cannot be tampered with since the exact mathematical problems would have to be re-executed by a computer, which is not possible (Kharpal). And secondly, even if someone was able to pass the mathematical problems, the key for each block relates back to the previous one and so on. Therefore, it would take a lot of time and resources to tamper with the blockchain. Finally, if the hacker got to the first ever block of the blockchain and was able to tamper it, the key for that block would change. The community who can see the blockchain can clearly see this change and can declare the blockchain invalid, erasing all the data of past transactions and documents (Karra).

Sources:

Karra, Akhil, and Kameswara Sarma Karra. 27 Nov. 2018.
